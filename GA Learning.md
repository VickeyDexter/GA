# GOOGLE ANALYTICS - GA4
 
GA IS A TABLE. 
	 
DATA DIFFERENTIATION : 
	 
    ○ DIMENSIONS --> TEXT
	EG., SOURCE, MEDIUM
	 
		○ METRICS --> NUMBERS
	EG.,SESSIONS, ENGAGEMENTS AND EVENT COUNT, PAGE VIEWS.
	 
	GA MEASUREMENT:
	 
	DONE USING COOKIES. 
	COOKIES ARE SMALL FILES STORED ON  BROWSERS. 
	COOKIES ARE IDENTIFIERS OF USERS OR BROWSERS.
	 
	NOTE: COOKIES DON'T STORE ANY INFO & HAVE LIMITED SIZE.

	RANDOM NUMBER + FIRST TIME STAMP FORMS CLIENT ID WHICH IS USER/BROWSER UNIQUE IDENTIFICATION. IT RECOGNIZES USER AS NEW OR EXISTING USER.
	 
	NOTE : EACH PAGE SHOULD CONTAIN GA JS CODE SNIPPET. WITH THIS GA IDENTIFIES THE USER AND TIMESTAMP.
	 
	PROCESS : 
	1] LAUNCH A WEBSITE
	2] BROWSER DOWNLOADS AND PARSES HTML CODE PART OF IT IS A GA CODE.
	3] GA CODE ASKS BROWSER FOR GA COOKIES AND IF YES, IT TAKES CLIENT ID AND KNOWS IT'S A RETURNING USER.
	4] IF NOT, GA CODE HAS TO CREATE A COOKIE AND TAKE A CLIENT ID FROM IT.
	5] GA WILL SEND INFO TO GA SERVERS.
	 
	GA SERVER CALL PROCESS:
  
	 WHAT INFO IS STORED IN GA COOKIES?
	--> ONLY USED ID
	 
	3.BASIC METRICS
	 
	HIGHEST ENITY IS USERS OR DEVICES.
	 
	ONE USERS CAN HAVE MULTIPLE SESSIONS AND VIEWS.
	 
	GA4 USER IDENTIFICATION METHODS :
	 
		1. USER ID : IDENTIFIED WITH EVERY HIT IN THE PAGE
		2. GOOGLE SIGNALS : GAVE PERMISSION TO GOOGLE AND GOOGLE CREATES ITS OWN USER ID FOR IDENTIFICATION.
		3. DEVICE ID : IDENTIFIER STORED IN THE COOKIE.
		4. MODELLING : GOOGLE COLLECTS DATA OF NON-CONSENTED USER ANONYMOUSLY.
	 
	PATH TO FIIND IDENTIFICATION METHODS IN GA : GA -> ADMIN : SCROLL DOWN -> REPORTING IDENTITY
	
	 
	NOTE : COMMON ONE IS BLENDED.
	 
	WE CAN CHANGE FROM BLENDED TO OBSERVED AND IT DOESN'T AFFECT THE DATA.
	 
	SESSIONS & ENGAGEMENT:
	--> SESSIONS & ENGAGED SESSIONS :
	 
	ENGAGED SESSIONS:WHEN USER VIEWS THE PAGE OR DOES ANY ACTIVITY IN THE PAGE.
	CONDITIONS : 
		1. IF SESSIONS > 10 SECONDS
		2. IF SESSIONS < 10 SECONDS BUT ANY ACTION IN THE PAGE OR CLICKED 2 OR MORE PAGES
		
		IF USER IS INACTIVE FOR MORE THAN 30 MINS, SESSION WILL BECOME INACTIVE.DEFAULT ADJUST SESSION TIMEOUT IS SET TO 30 MINS.
	 
	PATH :ADMIN -> PROPERTY SETTINGS -> DATA COLLECTION AND MODIFICATIONS -> DATA STREAMS-> MAIN TAB  ALL CLICK " >" --> SCROLL DOWN -> CONFIGURE TAG SETTINGS -> SCROLL MORE -> ADJUST SESSION TIMEOUT -> ADJUST TIMER FOR ENGAGED SESSIONS (DEFAULT 10 SECONDS) 	 
	 
	
	 
	ENGAGEMENT RATE VS BOUNCE RATE : 
	
	Engagement Rate = Engaged Session/Session  NOTE : User spends more than 10secs/Session
	
	Also,
	Engagement Rate = 1 - Bounce Rate  NOTE : Bounce Rate is user viewed only 1 page or not
	met with Engaged session condition.
	
	Bounce Rate + Engaged Rate = 100%.
	
	
	
	
	 
	
	ACTIVE USERS AND TOTAL USERS:
	
	Path : Acquisition -> User Acquisition
	
	Active Users means users with at least one Engaged Session.
	
	Total Users are all users of Google w/o the need for an engaged session.
